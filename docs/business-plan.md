# CryptoBooks — Business Plan

## Mission
Build the de facto financial documentation toolkit for freelancers paid in crypto.

## Problem
Freelancers who receive stablecoin payments have zero professional tools to:
- Generate receipts for income received
- Create bank-style statements for landlords, lenders, accountants
- Invoice clients with crypto payment links
- Organize transaction history for tax filing

Current workaround: screenshot Etherscan and pray.

## Market
- Crypto tax software market: $1.2B+, growing 20%+ annually
- Freelancers processing $1T+ in stablecoin payments
- ~35% of freelancers receive some income in crypto
- **Competition is weak**: existing tools (Koinly, CoinTracker) target *traders*, not *earners*

## Revenue Model

| Tier | Price | Features |
|------|-------|----------|
| Free | $0 | 5 receipts/statements/mo, basic branding |
| Pro | $19/mo | Unlimited everything, custom branding, CSV export |
| Business | $49/mo | Multi-user, white-label, API access, priority support |

## Unit Economics
- **COGS per user:** ~$0.01/mo (API calls, hosting)
- **Gross margin:** ~99%
- **Break-even:** 1 paying user
- **$10k/mo target:** ~520 Pro users OR 300 Pro + 100 Business

## Go-to-Market
1. **Crypto Twitter / X** — threads, memes, founder story
2. **Reddit** — r/ethdev, r/ethfinance, r/freelance
3. **Product Hunt** — launch with polished demo
4. **Indie Hackers** — build-in-public posts
5. **Agent ecosystem** — API-first for AI agent integrations

## Competitive Moat
- **Data quality:** Clean, labeled, categorized transaction history
- **Agent infrastructure:** Become the standard API for crypto income data
- **Community:** Build-in-public, freelancer-focused, not trader-focused

## Timeline
- **Month 1:** Receipt generator MVP (live)
- **Month 2:** Statements + dashboard
- **Month 3:** Invoices + payment links + launch
- **Month 4+:** API, agents, scale
